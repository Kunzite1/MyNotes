# 3 FreeRTOS的启动流程

## 3-1 任务创建

系统上电时，首先执行的是启动文件中的复位函数（汇编编写）`Reset_Handler`，函数最后会调用__main函数，在其中初始化系统的堆和栈，最后调用C中的main函数。

在main函数中，可以直接初始化外设资源，然后用`xTaskCreate()`函数直接创建任务。`xTaskCreate`函数会自动进行一系列初始化。

## 3-2任务调度

创建完任务后要开启调度器。使用开启调度器函数：`vTaskStartScheduler()`，其中会实现空闲任务和定时器任务。

### 3-2-1 SVC PendSV SysTick

Cortex-M3架构中，FreeRTOS为任务启动和切换提供了三个异常（异常是一种中断）：SVC、PendSV、SysTick。PendSV和SysTick的优先级配置为最低。

- SVC：用于任务启动。有些操作系统中，应用程序无法直接访问硬件，而是提供以下系统服务函数。SVC会发出堆系统服务函数的呼叫请求，以访问硬件。

- PendSV：用于任务切换，且是可挂起的。当有优先级高于他的中断在运行，PendSV就会延迟执行。

- SysTick：用于产生系统时钟节拍，提供时间片。

## 3-3 主函数

首先执行`SysTick()`和设置系统中断优先级分组。

然后进行外设模块初始化，比如LED、按键、传感器等等模块的初始化。

接着创建任务，和开启任务调度器。

启动任务调度器时，如果启动失败，会进入死循环。

```c
int main()
{
    SysTick_Init(72);
    NVIC_PriorityGroupConfig(NVIC_PriorityGroup_4);//设置系统中断优先级分组4
    LED_Init();
    USART1_Init(115200);

    //创建开始任务
    xTaskCreate((TaskFunction_t )start_task,            //任务函数
                (const char*    )"start_task",          //任务名称
                (uint16_t       )START_STK_SIZE,        //任务堆栈大小
                (void*          )NULL,                  //传递给任务函数的参数
                (UBaseType_t    )START_TASK_PRIO,       //任务优先级
                (TaskHandle_t*  )&StartTask_Handler);   //任务句柄              
    vTaskStartScheduler();          //开启任务调度
}
```
