# 1 任务的实现过程

## 1-1 定义任务栈

- 栈是单片机RAM内一段连续的内存空间。

- 有多少个任务就需要定义多少个任务栈。

- 任务栈就是一个预先定义好的全局数组，数据类型为StackType_t，大小由宏TASK1_STACK_SIZE来定义（默认128）。

## 1-2 定义任务函数

任务是一个独立的函数，函数主体无限循环，且不能返回。如：

```c
void task1_task(void *pvParameters)
{
   while(1)
   {
       // 要运行的代码 
   }
}
```

## 1-3 定义任务控制块

任务控制块就是一个结构体，在task.c中声明。其包含了任务的所有信息，比如栈指针、名称、形参等等。如：

```c
typedef struct tskTaskControlBlock
{                    
    volatile StackType_t *pxTopOfStack; /* 栈顶 */ 
    ListItem_t xStateListItem; /* 任务节点 */ 
    StackType_t *pxStack; /* 任务栈起始地址 */ 
    /* 任务名称， 字符串形式 */
    char pcTaskName[ configMAX_TASK_NAME_LEN ];
} tskTCB;
typedef tskTCB TCB_t;
```

## 1-4 列表

列表的初始化有一个专门的函数：`vListInitialise()`，在list.c中定义。

## 1-5 列表项

### 1-5-1列表项插入

列表项插入函数：`vListInsert()`。

### 1-5-2 列表项删除

删除函数：`uxListRemove()`

## 1-6 任务创建函数

函数①：`xTaskCreateStatic()`

函数②：`prvInitialiseNewTask()`

函数③：`pxPortInitialiseStack()`

## 1-7 就绪列表

创建好任务后，就要把任务添加到就绪列表中，表示任务已就绪，系统可以随时调度。

### 1-7-1 定义就绪列表

就绪列表实际就是一个List_t的数组。

数组大小取决于最大任务优先级的宏`configMAX_PRIORITIES`决定

### 1-7-2 就绪列表初始化

使用初始化函数：`prvInitialiseTaskLists()`

### 1-7-3 将任务插入到就绪列表

通过把任务控制块的`xStateListItem`这个节点插入到就绪列表来实现把任务插入到就绪列表。

<u>如果把就绪列表比作是晾衣架， 任务是衣服，那 xStateListItem 就是晾衣架上面的钩子， 每个任务都自带晾衣架钩子， 就是为了把自己挂在各种不同的链表中。</u>

## 1-8 调度器

这是操作系统的核心，主要功能就是实现任务的切换。全部在task.c中实现。

### 1-8-1 启动调度器

由函数：`vTaskStartScheduler()`来完成。此函数创建任务和配置定时器等等，随后调用函数`xPortStartScheduler()`来初始化调度器相关硬件资源，如滴答定时器、FPU单元、PendSV中断等。

在函数`xPortStartScheduler()`中又会调用`prvStartFirstTask()`以开始第一个任务：更新MSP值、产生SVC系统调用，然后去到 SVC 的中断服务函数里面真正切换到第一个任务。

### 1-8-2 任务切换

本质就是在就绪列表中寻找优先级最高的就绪任务，并去执行它。

由函数`taskYIELD()`完成任务切换。

PendSV中断服务函数`xPortPendSVHandler()`是真正实现任务切换的地方。
